# ユビキタス言語: 認証・認可コンテキスト

## User
- **定義**: システムにログインできる個人またはエンティティ。
- **属性**:
    - 一意な識別子 (ID)
    - メールアドレス (ログインIDとして利用)
    - ハッシュ化されたパスワード
- **責務**:
    - 自身のプロファイル情報を保持する。

## Identity
- **定義**: システムにおけるユーザーの「本人性」を表現する概念。認証プロセスをカプセル化する。
- **責務**:
    - 与えられた資格情報（メールアドレス、パスワード）が正当であるかを検証する (認証)。

## Role
- **定義**: ユーザーに割り当てることができる権限の集合。職務や役割に基づく。
- **例**: `admin` (管理者), `editor` (編集者), `viewer` (閲覧者)
- **責務**:
    - 複数の `Permission` をグルーピングする。

## Permission
- **定義**: システム内で実行できる、具体的かつ最小単位の操作。
- **例**: `create-post`, `edit-post`, `delete-post`
- **責務**:
    - 特定のアクションやリソースへのアクセス可否を表現する。
