# ユビキタス言語

このドキュメントには、`Authenticate` モジュール内で使用される主要なドメイン用語とその定義を記述します。

---
## 認証 (Authentication)
ユーザーが本人であることを確認するプロセス。システムへのアクセスを許可する前に実行される。

## 認可 (Authorization)
認証されたユーザーが特定のリソースや機能にアクセスする権限を持っているかを判断するプロセス。

## ユーザー (User)
システムを利用する個人またはエンティティ。

## 役割 (Role)
ユーザーに割り当てられる権限の集合。特定の役割を持つユーザーは、それに対応する操作を実行できる。

## 許可 (Permission)
特定の操作（例: データの読み取り、書き込み）を実行する権利。役割を通じてユーザーに付与される。

## 資格情報 (Credentials)
認証のためにユーザーが提供する情報。例: メールアドレス,パスワード、APIキー。

## セッション (Session)
ユーザーがシステムにログインしている期間。セッション中は、ユーザーの状態と認証情報が維持される。

## トークン (Token)
認証および認可のために使用されるデジタル署名付きのデータ。例: JWT (JSON Web Token)。

## パスワードリセット (Password Reset)
ユーザーがパスワードを忘れた場合に、新しいパスワードを設定できるようにするプロセス。

## 登録 (Registration)
新しいユーザーがシステムにアカウントを作成するプロセス。

## メール検証 (Email Verification)
登録時に提供されたメールアドレスが有効であることを確認するプロセス。

## 二段階認証 (Two-Factor Authentication, 2FA)
パスワードに加えて、別の認証要素（例: ワンタイムパスワード、生体認証）を要求することで、セキュリティを強化するプロセス。

## メールアドレスの変更 (Email Address Change)
ユーザーが登録済みのメールアドレスを別のメールアドレスに更新するプロセス。通常、変更の確認のために新しいメールアドレスへの検証が必要となる。
